# Работа с запросами и ответами
[Выше](/doc/py/util/index.md) | [Корень](/doc/index.md)

Утилиты для работы с запросами и ответами. Описано в [main.util](/main/util.py)

### `getArgumentOr(arr, name, default) -> V (str)|Any`
Возвращает значение аргумента в указанном массиве или стандартное значение при его отсутствии. Как правило имеет тип String
|Параметр |Тип                   |Описание                                                     |
|:--------|:---------------------|:------------------------------------------------------------|
|`arr`    |`dict[K(str), V(str)]`|Карта ключ-значение, в которой будет выполнен поиск аргумента|
|`name`   |`K (str)`             |Название параметра. Как правило имеет тип str                |
|`default`|`Any`                 |Стандартное значение параметра                               |



### `getGetOr(req, name, default) -> V(str)|Any`
Алиас для `getArgumentOr`, где `arr = req.GET`

### `getPostOr(req, name, default) -> V(str)|Any`
Алиас для `getArgumentOr`, где `arr = req.POST`

### `getSessOr(req, name, default) -> V(str)|Any`
Алиас для `getArgumentOr`, где `arr = req.session`

### `respond(status, data) -> HTTPResponse`
Подготавливает HTTPResponse с указанным статусом и данными
|Параметр|Тип  |Описание          |
|:-------|:----|:-----------------|
|`status`|`int`|HTTP-статус ответа|
|`data`  |`str`|Текст ответа      |



### `checkVars(var_list) -> bool`
Используется для проверки переменных (изначально GET, POST параметров) на соответствие ограничениям. Возвращает `True` в случае прохождения проверки всеми элементами, `False` в любом другом случае
|Параметр  |Тип            |Описание                                                               |
|:---------|:--------------|:----------------------------------------------------------------------|
|`var_list`|`list[var_def]`|Список var_def для проверки переменных (см. фиктивный тип var_def ниже)|



### `b64enc(rawstr) -> str`
Используется для кодирования строки в Base64
|Параметр|Тип  |Описание              |
|:-------|:----|:---------------------|
|`rawstr`|`str`|Строка для кодирования|



### `b64dec(b64str) -> str`
Используется для декодирования строки из Base64
|Параметр|Тип  |Описание                |
|:-------|:----|:-----------------------|
|`b64str`|`str`|Строка для декодирования|



### Фиктивный тип var_def
|Индекс|Тип/Значения           |Описание                                                                                                                                 |
|:-----|:----------------------|:----------------------------------------------------------------------------------------------------------------------------------------|
|0     |`str\|int\|float\|list`|Тип переменной для проверки ИЛИ список валидных значений                                                                                 |
|1     |`any(str)`             |Переменная для проверки                                                                                                                  |
|2     |`int\|float`           |Минимальное значение int/float, минимальная длина строки                                                                                 |
|3     |`int\|float`           |Максимальное значение int/float, максимальная длина строки                                                                               |
|4     |`re.Pattern\|str`      |REGEX-выражение для проверки строки (только когда 0 == `str`)                                                                            |
|Пример|                       |`[str, var, 1, 20, USERNAME_RE_MATCH]` - указывет: `var` - строка от 1 до 20 символов соответствующаяя выражению `util.USERNAME_RE_MATCH`|
